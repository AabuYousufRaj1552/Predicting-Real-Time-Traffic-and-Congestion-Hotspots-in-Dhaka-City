# Contributing to Traffic Prediction Project

Thank you for your interest in contributing to this traffic prediction project! This document provides guidelines for contributing.

## How to Contribute

### 1. Fork the Repository
- Fork the project on GitHub
- Clone your fork locally
- Create a new branch for your feature

### 2. Development Setup
```bash
# Clone your fork
git clone https://github.com/your-username/repository-name.git
cd repository-name

# Install dependencies
pip install -r requirements.txt

# Install Jupyter if not already installed
pip install jupyter
```

### 4. Pull Request Process

1. **Create a Branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

2. **Make Changes**
   - Write clean, documented code
   - Follow the existing project structure
   - Test your changes thoroughly

3. **Commit Changes**
   ```bash
   git add .
   git commit -m "Add: descriptive commit message"
   ```

4. **Push and Create PR**
   ```bash
   git push origin feature/your-feature-name
   ```
   - Open a pull request on GitHub
   - Provide a clear description of changes
   - Reference any related issues

### 5. Code Style Guidelines

#### Python Code
- Follow PEP 8 style guidelines
- Use meaningful variable names
- Add docstrings to functions
- Keep functions focused and small

#### Jupyter Notebooks
- Clear cell organization
- Markdown explanations between code sections
- Remove output before committing (optional)
- Include visualizations with proper titles and labels

#### Documentation
- Update README files when adding features
- Include code comments for complex logic
- Provide usage examples

### 6. Dataset Contributions

If you have additional traffic data:
- Ensure proper licensing and permissions
- Follow the existing data structure
- Include data description and source
- Consider privacy and ethical implications

### 7. Testing

- Test notebooks run without errors
- Verify visualizations display correctly
- Check that new models follow the established pattern
- Validate results make sense

### 8. Communication

- Be respectful and constructive
- Ask questions if unclear about requirements
- Respond to feedback promptly
- Help others in discussions

## Project Structure Guidelines

When adding new content:
- Place model-specific code in appropriate directories
- Use consistent naming conventions
- Add README files for new directories
- Update main README with significant changes

## Getting Help

- Check existing issues and documentation
- Ask questions in new issues
- Tag maintainers for urgent matters
- Join project discussions

## Recognition

Contributors will be acknowledged in:
- README contributor section
- Release notes for significant contributions
- Project documentation
